* Audio Unit Hosts
SooperLooper will run ok in *ardour* as an output plugin. Works on *arm*. =Very= annoying. Sometimes works, sometimes doesn't, meanwhile *Ardour* is constantly nagging about payment.
Sadly Audacity fucks up trying to do the same thing. Sometimes it works, usually *SooperLooper* wont start.
Havent tried it in *MPC Beats* yet. Not sure it even does *VST/AU* plugins.

** AU-Lab
will run *sooperlooper* but also seem to suffer from the same [[https://github.com/essej/sooperlooper/issues/43][isthusue]] and [[https://github.com/essej/sooperlooper/issues/32][issue]] (the workaround doesn't work)
Also *AULab* doesn't request access to the microphone (necessary for all sound inputs). Starting from *iTerm* gets around this
** [[https://github.com/kushview/element][Element]] 
Doesn't handle *SooperLooper* well. Looks like the same problems as elsewhere, but *Element* keeps trying to reconnect and is unresponsive until it gives up.
Works well with *Mobius* with no apparant timing problems unlike *VCV*, even without tweaking block size and sample rate. Also handles *GuitarRig*
** VCV
Has timing problems. Might be able to sort it out by adjusting input/output latencies.
** [[https://github.com/falkTX/Carla/][Carla]]
Open source plugin host. Handles *Mobius* well, but there is a problem getting midi in to *GuitarRig*
** [[https://github.com/DISTRHO/Cardinal/][Cardinal]]
*CardinalNative* Fully open source version of VCV rack

* Setup
*VCV* seems like the way to go. Either as a host or as a plugin for whatever looper plugin works best.
- If it is to be the host then I will need to buy the *VCV* host plugin. Bought it...
- If it is to be a plugin then I will need to buy the full version of *VCV*
The first is probably the best for a number of reasons:
- Based on open source *VCV*
- Likely to be usung multiple synths etc which can be done in *VCV* but not *Guitar Rig* probably don't want multiple instances of a processor/memory hungry app as a plugin
- Its cheaper
Why not?
- *Reason* might be a good choice as the base system. Quicker to get moving with various synths. But using *Reason* as a plugin would be ok, and if it proves to be too resource heavy to have multiple instances then anything *Reason* can do can be rebuilt in *VCV*, which would be lots of fun and highly educational.

=VCV host= and =Mobius= it is then. Maybe try =LogeLoop= if mobius has problems.
